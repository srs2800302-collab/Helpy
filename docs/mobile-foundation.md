# Fixi Mobile Foundation

## Current stack
- Flutter
- flutter_riverpod
- go_router
- dio
- intl

## Implemented
- app structure
- router
- localization
- auth controller
- auth api client
- splash/login/verify/select-role/home screens

## Current limitations
- token storage is in-memory only
- router redirect logic is minimal
- no persistent session restore across app restarts
- no jobs/profile integration yet

## Next step
- persistent token storage
- route redirect rules
- categories fetch
- job creation screens
